# KTX 수동 빈자리 확인 도우미

개인 사용을 위한 **로컬 웹 기반 KTX 빈자리 확인 도우미**입니다. 이전 버전처럼 Playwright/자동화 브라우저로 KORAIL을 직접 조작하지 않습니다. KORAIL은 평소 쓰는 브라우저에서 직접 로그인하고 직접 조회한 뒤, 조회 결과 텍스트만 이 도우미에 붙여 넣어 조건에 맞는 빈자리가 있는지 확인합니다. 모든 표가 매진인 상황에서 계속 새로고침해야 하는 목적을 위해, 자동 조작 대신 **수동 새로고침 타이머/알림/반복 판독 흐름**을 제공합니다.

## 왜 자동화 방식을 버렸나요?

KORAIL 로그인 화면에서 아래와 같은 메시지가 나올 수 있습니다.

```text
원활한 서비스를 위해 안정적인 환경에서 이용해 주시기 바랍니다.
매크로 등 미허가 도구 사용 시 이용이 제한될 수 있습니다.
CODE : -8003
```

이 메시지는 자동화 브라우저, 반복 조회, 비정상 접속 패턴 등이 KORAIL의 매크로 탐지에 걸렸을 때 나타날 가능성이 큽니다. 로그인만 사람이 직접 하더라도 Playwright가 띄운 브라우저 자체가 자동화 환경으로 보이면 로그인 전에 차단될 수 있습니다.

그래서 현재 버전은 아래 기능을 모두 제거했습니다.

- Playwright 사용 안 함
- 자동 로그인 안 함
- 자동 조회 안 함
- KORAIL 화면 자동 새로고침 안 함
- 자동 예약 버튼 클릭 안 함
- 결제 화면 자동 진입 안 함

대신 **KORAIL은 사용자가 직접 조작**하고, 이 프로그램은 **반복 확인할 타이밍을 알려주며 복사해 온 조회 결과 텍스트를 판독하는 보조 역할**만 합니다.

## 초보자용 전체 사용 설명서

아래 순서대로 천천히 따라 하면 됩니다.

### 1단계: Node.js 설치하기

이 프로그램은 Node.js로 실행합니다. 이미 설치되어 있다면 이 단계는 건너뛰어도 됩니다.

1. 웹 브라우저로 <https://nodejs.org/> 에 접속합니다.
2. **LTS**라고 표시된 버전을 다운로드합니다.
3. 설치 파일을 실행합니다.
4. 설치 화면에서는 특별히 바꾸지 말고 `Next`를 눌러 기본값으로 설치합니다.
5. 설치가 끝나면 터미널을 새로 엽니다.

설치 확인:

```bash
node -v
npm -v
```

버전 숫자가 나오면 설치가 된 것입니다.

### 2단계: 프로젝트 폴더로 이동

터미널에서 이 프로젝트 폴더로 이동합니다.
두 명령 모두 버전 숫자가 나오면 성공입니다. 예를 들면 `v22.0.0`, `10.0.0`처럼 표시됩니다.

### 2단계: 터미널 열기

운영체제별로 아래처럼 터미널을 엽니다.

#### Windows

1. 시작 메뉴를 엽니다.
2. `PowerShell` 또는 `명령 프롬프트`를 검색합니다.
3. 실행합니다.

#### macOS

1. `Command + Space`를 누릅니다.
2. `Terminal` 또는 `터미널`을 검색합니다.
3. 실행합니다.

### 3단계: 프로젝트 폴더로 이동하기

터미널에서 이 프로젝트가 있는 폴더로 이동해야 합니다.

예시:

```bash
cd /workspace/korail
```

내 PC에서 다른 폴더에 받았다면 해당 폴더 경로로 이동하세요.

### 3단계: 프로그램 실행

이 버전은 외부 npm 패키지를 사용하지 않으므로 `npm install` 없이도 실행할 수 있습니다.
내 컴퓨터에 다른 위치에 받았다면 해당 위치로 이동합니다. 예를 들어 다운로드 폴더 안에 있다면 아래처럼 될 수 있습니다.

```bash
cd ~/Downloads/korail
```

정상적으로 이동했는지 확인하려면 아래 명령을 입력합니다.

```bash
pwd
```

현재 위치가 `korail` 폴더로 끝나면 됩니다.

### 4단계: 필요한 프로그램 설치하기

프로젝트 폴더 안에서 아래 명령을 한 번만 실행합니다.

```bash
npm install
```

설치가 끝나면 `node_modules`라는 폴더가 생깁니다. 이 폴더는 직접 건드리지 않아도 됩니다.

만약 Playwright 브라우저가 없다는 오류가 나면 아래 명령을 추가로 실행하세요.

```bash
npx playwright install chromium
```

### 5단계: 프로그램 실행하기

프로젝트 폴더 안에서 아래 명령을 실행합니다.

```bash
npm run dev
```

아래와 비슷한 문구가 나오면 정상입니다.

```text
KORAIL manual seat helper running at http://localhost:3001
```

### 4단계: 대시보드 열기

브라우저 주소창에 아래 주소를 입력합니다.
성공하면 터미널에 대략 아래와 비슷한 문구가 나옵니다.

```text
KORAIL helper running at http://localhost:3001
```

이 터미널 창은 프로그램이 실행되는 동안 닫지 마세요. 닫으면 프로그램도 종료됩니다.

### 6단계: 대시보드 열기

웹 브라우저 주소창에 아래 주소를 입력합니다.

```text
http://localhost:3001
```

### 5단계: KORAIL에서 직접 로그인/조회

1. 대시보드의 **KORAIL 직접 열기** 버튼을 누릅니다.
2. 평소 쓰는 브라우저에서 KORAIL에 직접 로그인합니다.
3. KORAIL 화면에서 아래 조건 중 원하는 조건으로 직접 조회합니다.
   - 2026-05-23 광명 → 부산 09:00 ~ 13:00
   - 2026-05-25 부산 → 광명 09:00 ~ 15:00
4. 조회 결과 화면이 나오면 그 화면에서 전체 선택 후 복사합니다.
   - Windows: `Ctrl + A` → `Ctrl + C`
   - macOS: `⌘ + A` → `⌘ + C`

### 6단계: 매진일 때 반복 확인 타이머 시작

모든 탑승권이 매진이라 계속 새로고침해야 할 때 사용하는 기능입니다. 단, 프로그램이 KORAIL을 자동으로 새로고침하지는 않습니다. 자동 새로고침/자동 클릭은 매크로로 오인될 수 있으므로, 이 앱은 **언제 직접 새로고침할지 알려주는 타이머** 역할을 합니다.

1. 대시보드의 **수동 새로고침 도우미** 영역으로 갑니다.
2. 반복 확인할 조건을 선택합니다.
3. 확인 간격을 정합니다. 기본값은 35초이고, 너무 빠른 반복을 피하려고 최소 20초 아래로는 내려가지 않게 했습니다.
4. 필요하면 **브라우저 알림 허용**을 누릅니다.
5. **반복 확인 시작**을 누릅니다.
6. 타이머가 `지금 확인` 상태가 되면 KORAIL 탭으로 돌아가 사용자가 직접 새로고침 또는 조회 버튼을 누릅니다.
7. 다시 조회된 화면을 복사해서 아래 붙여넣기 영역에서 확인합니다.

### 7단계: 조회 결과 붙여 넣고 빈자리 확인

1. 대시보드의 **조회 결과 붙여넣기** 입력칸에 복사한 텍스트를 붙여 넣습니다.
2. 확인할 조건을 선택합니다.
3. **빈자리 확인** 버튼을 누릅니다.
4. 빈자리가 없으면, 체크된 옵션에 따라 다음 수동 새로고침 타이머가 자동으로 다시 시작됩니다.
5. 결과가 `예약 가능 후보`로 나오면 타이머가 멈추고 알림음/브라우저 알림이 뜹니다. KORAIL 화면으로 돌아가 직접 예약 버튼을 누릅니다.

> 주의: 이 도우미는 붙여 넣은 텍스트를 추정 판독합니다. KORAIL 화면에 예약 버튼이 직접 보이면 이 도우미 결과보다 KORAIL 실제 화면을 우선하세요.

### 8단계: 종료

1. 브라우저에서 대시보드 탭을 닫습니다.
2. 터미널에서 `Ctrl + C`를 눌러 서버를 종료합니다.

## GitHub에서 “This branch has conflicts”가 보이는 경우

GitHub 화면의 **This branch has conflicts that must be resolved** 문구는 프로그램 실행 오류가 아니라, 현재 작업 브랜치와 GitHub의 기준 브랜치가 같은 파일을 서로 다르게 수정해서 자동 병합을 못 한다는 뜻입니다. 스크린샷에 보인 충돌 파일은 `README.md`, `package.json`, `public/app.js`, `public/index.html`, `public/styles.css`, `src/config.js`, `src/server.js`입니다.

초보자라면 GitHub 웹 편집기에서 충돌 표시를 하나씩 고치기보다, 아래 방식처럼 **현재 브랜치를 최신 기준 브랜치 위에 다시 올린 뒤 충돌 파일은 이 프로젝트 버전으로 유지**하는 편이 가장 단순합니다.

```bash
git fetch origin
git checkout 작업브랜치이름
git rebase origin/main
```

충돌이 난 파일이 표시되면, 이 프로젝트의 수동 빈자리 확인 도우미 버전을 그대로 쓰기 위해 아래 명령을 실행합니다.

```bash
git checkout --theirs README.md package.json public/app.js public/index.html public/styles.css src/config.js src/server.js
git add README.md package.json public/app.js public/index.html public/styles.css src/config.js src/server.js
git rebase --continue
```

만약 기준 브랜치 이름이 `main`이 아니라 `master`라면 첫 번째 명령의 `origin/main`을 `origin/master`로 바꾸면 됩니다. `rebase` 도중에는 현재 작업 브랜치의 변경분이 `--theirs`로 표시되므로, 위 명령은 충돌 파일을 이 프로젝트 버전으로 선택합니다. 충돌 해결 뒤에는 아래 명령으로 문법 검사를 한 번 실행하세요.

```bash
npm run check
```

그 다음 GitHub에 다시 올립니다.

```bash
git push --force-with-lease
```

> 참고: 이 저장소 버전은 KORAIL 자동 조작을 하지 않는 수동 도우미입니다. 충돌을 해결할 때 Playwright나 자동 클릭 코드가 다시 섞이지 않도록 위 파일들은 현재 버전을 유지하는 것을 권장합니다.

## CODE -8003 메시지가 나온 경우

아래 순서로 진행하세요.

1. Playwright/자동화 브라우저가 열려 있다면 모두 닫습니다.
2. 기존 자동화 버전에서 실행하던 터미널도 `Ctrl + C`로 종료합니다.
3. 평소 쓰는 일반 브라우저(Chrome, Edge, Safari 등)를 새로 열어 KORAIL에 직접 접속합니다.
4. 그래도 막히면 잠시 기다렸다가 다시 시도합니다.
5. 정상 환경에서도 계속 막히면, 오류 화면의 `CODE`와 `ID`를 KORAIL 고객센터에 전달해 문의하는 것이 안전합니다.

이 프로젝트는 차단을 우회하거나 탐지를 피하는 기능을 제공하지 않습니다. 자동화로 오인될 수 있는 동작을 없애고, 수동 조회 결과를 정리해 보는 방식으로만 동작합니다.

## 자주 묻는 질문

### 매진이면 자동으로 계속 새로고침하나요?

아니요. KORAIL 페이지를 프로그램이 자동 새로고침하지 않습니다. 대신 **수동 새로고침 도우미**가 35초 같은 간격으로 알림을 주면, 사용자가 정상 브라우저에서 직접 새로고침/조회 버튼을 누르는 방식입니다. CODE -8003 같은 차단을 피하려고 자동 조작은 넣지 않았습니다.

### 이제 자동으로 예약 버튼을 눌러주지 않나요?

네. 현재 버전에서는 누르지 않습니다. KORAIL의 매크로 탐지에 걸리지 않도록 자동 클릭 기능을 제거했습니다.

### 그럼 이 프로그램은 무엇을 해주나요?

KORAIL 조회 결과 화면의 텍스트를 붙여 넣으면, 설정된 날짜/구간/시간대 안에서 KTX 예약 가능 후보가 있는지 빠르게 찾아줍니다. 또한 매진일 때 반복 확인할 수 있도록 타이머, 알림음, 브라우저 알림을 제공합니다.

### 붙여 넣었는데 판독이 안 된다고 나와요

KORAIL 화면 구조나 복사된 텍스트 형태에 따라 자동 판독이 어려울 수 있습니다. 이때는 KORAIL 결과 화면에서 브라우저 찾기(`Ctrl + F` 또는 `⌘ + F`)로 `예약`, `예매`, `특실` 같은 단어를 직접 검색해 보세요.

### `npm install`을 해야 하나요?

현재 버전은 외부 패키지가 없어서 필수는 아닙니다. `npm run dev`만 실행하면 됩니다.

### 화면을 새로고침하면서 계속 자동으로 확인해주면 안 되나요?

그 기능은 일부 환경에서 매크로로 오인될 수 있어 넣지 않았습니다. 이 버전의 원칙은 **사용자가 직접 조회하고, 프로그램은 텍스트만 판독**하는 것입니다.

### 7단계: KORAIL 로그인하기

1. 대시보드에서 **로그인 브라우저 열기** 버튼을 누릅니다.
2. 자동화용 Chromium 브라우저가 새로 열립니다.
3. 열린 KORAIL 화면에서 직접 로그인합니다.
4. 로그인 후에는 이 브라우저 창을 닫지 말고 그대로 둡니다.

아이디와 비밀번호는 프로그램에 저장하지 않습니다. 로그인 상태만 `data/browser-profile` 폴더에 저장되어 다음 실행 때 이어질 수 있습니다.

### 8단계: 좌석 감시 시작하기

대시보드에는 두 작업 카드가 보입니다.

- **2026-05-23 광명 → 부산 09:00 ~ 13:00**
- **2026-05-25 부산 → 광명 09:00 ~ 15:00**

원하는 작업의 **감시 시작** 버튼을 누릅니다.

프로그램은 약 30초마다 KORAIL 페이지를 조회합니다. 화면의 상태와 로그에서 현재 동작을 확인할 수 있습니다.

### 9단계: 좌석이 발견되면 직접 마무리하기

좌석이 발견되면 프로그램이 자동화 브라우저에서 가능한 예약/예매 버튼을 누릅니다. 이후 결제 직전 또는 본인 확인/보안 확인 화면까지 최대한 이동합니다.

그 다음부터는 직접 진행해야 합니다.

- 보안 확인
- 본인 인증
- 결제 수단 선택
- 최종 결제

좌석이 발견되면 소리/화면 로그를 확인하고 자동화 브라우저로 바로 이동하세요.

### 10단계: 프로그램 종료하기

사용을 끝내려면 아래 순서로 종료합니다.

1. 대시보드에서 실행 중인 작업의 **중지** 버튼을 누릅니다.
2. 자동화 브라우저를 닫습니다.
3. 터미널에서 `Ctrl + C`를 누릅니다.

## 자주 생기는 문제 해결

### `npm` 또는 `node` 명령을 찾을 수 없다고 나와요

Node.js가 설치되지 않았거나, 설치 후 터미널을 새로 열지 않은 경우입니다.

1. Node.js LTS 버전을 다시 설치합니다.
2. 터미널을 완전히 닫았다가 다시 엽니다.
3. `node -v`, `npm -v`를 다시 실행합니다.

### `npm install` 중 오류가 나요

인터넷 연결이 필요합니다. 회사/학교/공공 와이파이에서는 npm 다운로드가 막힐 수 있습니다.

가능하면 집 인터넷이나 개인 핫스팟에서 다시 시도하세요.

### 브라우저가 안 열려요

아래 명령을 실행한 뒤 다시 `npm run dev`를 실행해 보세요.

```bash
npx playwright install chromium
```

### 대시보드가 안 열려요

먼저 터미널에서 `npm run dev`가 계속 실행 중인지 확인하세요. 실행 중이라면 브라우저 주소창에 아래 주소를 정확히 입력합니다.

```text
http://localhost:3001
```

다른 프로그램이 3001번 포트를 사용 중이면 서버가 시작되지 않을 수 있습니다. 그 경우 터미널 오류 메시지를 확인해야 합니다.

### 로그인했는데 다시 로그인하라고 나와요

KORAIL 세션이 만료되었을 수 있습니다.

1. 자동화 브라우저에서 다시 로그인합니다.
2. 대시보드에서 작업을 다시 시작합니다.

### 좌석이 있는데도 못 잡았어요

가능한 원인은 여러 가지입니다.

- 클릭하는 순간 다른 사람이 먼저 예약함
- KORAIL 화면 구조가 바뀜
- 보안 확인/CAPTCHA가 나타남
- 예약 가능한 좌석이 입석/자유석이라 프로그램이 제외함
- 네트워크가 느려 조회가 늦어짐

이 프로그램은 예매 성공을 보장하는 도구가 아니라, 좌석 확인과 예약 화면 진입을 도와주는 보조 도구입니다.

### 오류 스크린샷은 어디에 저장되나요?

자동화 중 오류가 나면 `data/screenshots` 폴더에 화면 캡처가 저장될 수 있습니다. 문제가 생겼을 때 이 이미지를 보면 어느 화면에서 멈췄는지 확인할 수 있습니다.

## 사용 시 주의사항

- KORAIL 약관과 이용 정책을 지켜서 사용하세요.
- 너무 짧은 간격으로 반복 조회하지 마세요. 기본값 30초를 권장합니다.
- CAPTCHA, 본인확인, 결제, 보안 확인은 자동화하지 않습니다.
- 이 프로그램은 내 컴퓨터에서만 쓰는 개인용 도구로 만들었습니다. 서버에 올려 여러 사람이 쓰는 용도는 아닙니다.
- 좌석이 보이는 순간 다른 사용자가 먼저 가져갈 수 있으므로 예약 성공을 보장하지 않습니다.

## 개발자용 명령어

문법 오류만 빠르게 확인하려면 아래 명령을 실행합니다.

```bash
npm run check
```

서버 실행:

```bash
npm run dev
```
